# Omega Resume - Workspace Agent Rules

## Project Structure
- `.` (Root): Go backend repository.
- `./frontend`: React frontend application.

## General Guidelines
- Always write clean, maintainable, and well-documented code.
- Follow Go standard formatting (`gofmt`) and idiomatic Go practices for the backend.
- Use functional components and React hooks for the frontend.
- Maintain separation of concerns between backend (API) and frontend (UI).
- Use `requirement.md` as the single source of truth for project phases and requirements.
- Before making significant architectural changes, propose an implementation plan.

## Backend (Go)
- Ensure all endpoints are properly tested.
- Keep handlers small and delegate logic to services.

## Frontend (React)
- Create reusable UI components.
- Ensure the application is responsive and follows modern design aesthetics.
